Health Information Processing — ARC

What this is?

One-file, pure JS + JSON demo for health-info processing using ARC: Answer • Reason • Check.

  • Data — patients, encounters, documents; observations like consent, PHI, labs, coding, duplicates.
  • Policies — declarative JSON (if atoms with optional "not": true ⇒ then atoms).
  • Answer — newly derived facts (only strongest priority per patient).
  • Reason Why — mathematical-English derivations.
  • Check — ≥10 JSON assertions over the full fact set.
  • Health Info Summary — per-patient best priority & tasks (Blocked > High > Normal > Low).

Engine: small forward-chainer with unification, subclassing (subClass), transitive partOf, observation→patient mapping, and negation-as-failure for "not": true.
